24 Purge old caches and limit dynamic cache size

When we change version number of SW we create a new cache to keep ins sync.

The SW does not know what is the latest version and as it searches from start and will probably use older caches.

We now purge old cache versions. This is best done in the ACTIVATE event as new caches can be formed in the INSTALL event.

We also create a function to trim the number of entries in the dynamic cache to avoid any possible storage issues.

skipWaiting and clients.claim used.
