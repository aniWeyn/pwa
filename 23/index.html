23 Dynamic cache and Fallback page

We have added a method to clone responses from network and place in DYNAMIC CACHE. The response is a stream and can only be used once so we must make a clone (copy).

If the page is not in cache or available from the network then the CATCH error occurs. We can server up the FALLBACK page.

We have added .skipWaiting to programatically activate the new SW.

We can determine the nature of the request, (HTML, CSS, image etc) and then serve back various FALLBACK pages.

Different fallback page for sample01.html than others.
