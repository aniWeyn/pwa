20 Demonstrate skipWaiting and when REGISTER/INSTALL/ACTIVATE are fired.

We have a very basic SW that just console.logs INSTALL and ACTIVATE. It also console.logs the FETCH event with the URL requested.

Refresh the page to see the FETCH in console.log

Lets comment out the FETCH console.logs

If we change one byte of a SW then the browser knows a new version exists and will install it.

We see nothing changes. Why?

As a tab is currently being controlled by the old SW, by default the new SW is installed but kept in waiting until all tabs with old SW are terminated.

If we look at DEV TOOLS >APPLICATION > Service Worker, we can see this.

We can either click skipWaiting or when developing tick the UPDATE ON RELOAD.

Once we click skipWaiting it will be activated, console log will show this and the new SW will operate.

Users won't know about this so there are ways to do it programatically. We look at this later. For now, we can understand the lifecycles of SW.
